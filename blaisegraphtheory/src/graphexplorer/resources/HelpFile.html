Help for GraphExplorer

Graphical User Interface

Generating New Graphs

Currently, the application has support for generating empty, circle, star, wheel, and complete graphs. In addition, a uniform random graph may be generated (directed or undirected) using either (i) a number of nodes and probability of edge, or (ii) a number of nodes and a number of edges. A preferential attachment graph may be generated, using a small initial uniform random graph as a seed, in which each vertex is added with either (i) a fixed number of connections, or (ii) a variable number of connections (with a user-specified probability distribution).

File Types

Currently supported file types include:
  • Edge list format: each line consists of a pair of integers indicating the edge between two nodes.
  • Pajek format (.net): vertex declarations (with positions), and a variety of input formats for edges, including edge lists and adjacency matrices.
  • Extended pajek format (.netx): pajek format with additional custom file information
  • UCINet format (.dat): supports more general data structures
Note that only part of these formats is currently supported! In the future, we hope to add support for both GraphML and DyNetML formats.

Visualization Features

Many visual settings may be adjusted using the property panel at left.

Longitudinal Graphs

This application supports longitudinal graphs, which change over time. A sample longitudinal graph may be created using the "preferential attachment" random graph generator, and selecting the longitudinal option. Special features included with the longitudinal view include:
  1. A slider below the main graph that displays the time frame for the longitudinal graph, which may be adjusted dynamically. If a dynamic layout is active, the layout automatically reconfigures for the new graph.
  2. A chart displaying the value of the selected metric over time. For this to be active, one or more nodes must be highlighted: use the Layout -> Highlight Nodes to select these nodes.

Layout Methods

Supported layouts are summarized here.

Circular Layout

Places nodes around a circle.

Random Layout

Places nodes at random within a box.

Spring layout

Uses simulated physics to generate ideal locations for nodes and edges. Forces include (i) a repulsive force between each pair of vertices, (ii) an attractive force between each pair of connected vertices, and (iii) a global force ensuring components stay close to the middle of the screen.

Metrics

Currently, one metric at a time may be displayed along with the graph. The metric displays in several ways: (i) the node sizes on the main plot are adjusted to have areas proportional to that node's metric value, (ii) the actual metric values are displayed on the main graph table, and (iii) a chart & table of the distribution of metric values is shown at bottom.
